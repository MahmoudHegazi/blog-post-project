HTML Intro

What is HTML?

HTML, which refers to Hypertext Markup Language, was created in 1993 by Sir Tim Berners-Lee. We can say that HTML is the selection of any webpage. Almost all web browsers use HTML, but note that you can create a custom browser that works with different markup languages instead of HTML, though you will likely lose all the benefits of the current web. We will discuss that in "Simple Idea About HTML and Browser."

Understand HTML:

HTML contains elements, and there are two types of elements in HTML: block elements and inline elements. Block elements are elements that take up the full width of the page, such as "DIV," "H1," and "P," while inline elements are elements placed next to each other on the page, so they take the width of their content. HTML elements must contain at least an opening tag and can contain attributes defined in the opening tag. Usually, HTML elements will contain closing tags unless they are void elements.

HTML contains elements, and there are three types of elements, which are:

  • Void Elements:
    • These elements are considered inline elements. They cannot have child elements or text within them, and they only have a start tag without a closing tag. Some void elements can contain text within them, but this text is included in the element attribute.
  • Raw Text Elements:
    • This special type of element cannot contain child elements within it, but it can contain attributes and text or special text such as "SCRIPT," "PRE," and "CODE." Browsers may format the text of these elements in different ways, as seen with "PRE" for example.
  • Normal Elements:
    • Normal elements are block elements that can contain attributes and child elements inside them, and they have opening and closing tags. They can also contain text, such as "DIV," "H1," "P," "Button," and "A."

Simple Idea About HTML and Browser:

Why is HTML a markup language? Because the text within an HTML file is converted into elements based on the HTML element rules for each element. Each element extracted from the text is converted into a static-looking element in the browser. For example, if you provide the valid markup text for the "DIV" element in the HTML file, which starts with an opening tag with the word "div" and ends with a closing tag with the word "/div," the browser will take this text and convert it into a static-looking element. So, markup means that you provide a text with some writing rules, and the program or browser takes it and renders it in a static known look, repeating this for all content in the HTML file. At the end, all "div" elements will look similar unless other languages are used, such as CSS, or if no style is applied as an attribute to the element. In simple terms, markup means that whenever you find "x," you draw it in your program as "shape_x," and so on for all other elements.

How to Create a Simple Markup Language such as HTML:

As a quick example, if we want to create a markup language such as HTML, we will need a computer program to read that language or text included in the file. According to the valid syntax of the elements, it will programmatically create those elements and repeat the process. For example, if we use Python as a programming language to create a simple browser using Tkinter, a simple library that can program the elements in desktop apps, we ignore the fact that browsers deal with HTTP protocols and need custom rules. We can say that the new language text will be contained in a text file within our Python program or provided as an API response to the Python program. The first step is to define the syntax of the text of our markup language and the elements we can draw using the Python program and Tkinter. Then users can create the text file that includes the valid syntax of the new markup language.

The second step, after the browser gets the file containing the new markup language (Python), will be for the programming language to try to read the provided text file, which includes the new HTML or our new markup language, which is actually a text but has specific rules or syntax when created. Then, Python can use a search technique or methods such as "REGEX" to search for the syntax of each element, and for each found element, it will write equivalent programming code that is needed to create the element in the same way as provided in the text but in Tkinter using programming methods. For example, "H1" will create a bigger title element in Tkinter, and so on. At the end, we can say that HTML is an easy way for people who do not have access to browsers or cannot program to provide text that browsers can search for and convert into programming methods that ultimately draw the page as the creator intended. Instead of wasting time creating one page, it is just about learning the markup language syntax and providing text with the name of the elements so that the browser can draw each element programmatically in the app, which in our case is the Tkinter app.

In the end, the Tkinter app will be able to convert any equal pages to the same looking elements. For example, any file received that contains only "h1" will be rendered as a header with specific size and uniformity across all instances. That is also why some browsers may differ when dealing with HTML, as each one processes the file using its own program. However, top browsers will follow a set of major guidelines, so changes will not be very big or noticeable; the differences will mainly be seen in some high-level new styles or elements, and not in the sizes of "h1," etc., or even in its support.
